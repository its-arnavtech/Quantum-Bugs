# Quantum Mutation Dataset

This dataset is generated by operator-driven templates so it can scale to hundreds of first-order mutants and a smaller higher-order set without hand-editing every file.

## Layout

- `seeds/`: baseline Qiskit-first seed programs
- `mutants/<family>/`: generated mutants grouped by algorithm family
- `metadata/mutants.csv`: flat analysis table
- `metadata/mutants.json`: structured metadata records
- `metadata/mutation_catalog.json`: mutation taxonomy plus generated operator inventory
- `metadata/metadata_schema.json`: field schema for downstream tooling
- `scripts/generate_mutants.py`: regenerates the dataset
- `scripts/validate_mutants.py`: parses and optionally executes seeds and mutants
- `scripts/run_mutants.py`: runs a selected slice of generated mutants

## Design Notes

- Qiskit is the primary target stack.
- Mutants are generated from explicit mutation sites rather than random edits.
- Each first-order mutant applies one primary mutation operator to one site.
- Higher-order mutants combine two first-order mutations from the same seed.
- Some mutants are intentionally equivalent candidates or intentionally non-runnable.

## Regeneration

```powershell
python quantum_mutation_dataset/scripts/generate_mutants.py
python quantum_mutation_dataset/scripts/validate_mutants.py
```
